# Project Context & Tech Stack
- **Project Name:** İrma Global Forwarding CRM
- **Type:** Web-Based Logistics ERP/CRM
- **Tech Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, Shadcn/UI (Design System), Supabase (Auth, DB, Realtime), Lucide React (Icons).
- **State Management:** React Query (Tanstack Query) + Zustand.
- **Form Handling:** React Hook Form + Zod.

# Design & UX Principles
- **Aesthetic:** Clean, minimalist, "Apple-esque" but data-dense. Use whitespace effectively. Not cluttered.
- **Behavior:** IBM Maximo-like lifecycle management. States (Draft -> Active -> Completed) are strictly controlled.
- **Responsive:** Mobile-first compatible but optimized for Desktop dashboards.
- **Visual Feedback:** Use toast notifications for success/error. Use skeleton loaders for data fetching.

# Core Business Rules (STRICT ENFORCEMENT)
1.  **Position Lifecycle:** A position CANNOT be closed without specific validation triggers (See "Document Lock").
2.  **Document Lock:**
    -   Status "Yola Çıktı" (Departed) IS BLOCKED unless: Driver License, License Plate, Insurance, Transport Contract are uploaded.
    -   Status "Kapandı" (Closed) IS BLOCKED unless: Sales Invoice (Navlun) exists, Purchase Invoice (Tedarikçi) exists, POD (CMR/Teslim Evrağı) is uploaded.
3.  **Supplier Logic:**
    -   Suppliers CANNOT be paid or invoiced without a pre-generated "Work Order Reference Number".
    -   Payment due dates are auto-calculated based on CMR delivery date + Term days.
4.  **Multi-Currency:** System must handle TRY, USD, EUR, RUB. Display profitability in a base currency (e.g., USD or TRY) but store original transaction currency.

# Coding Standards
-   Use Functional Components with strict TypeScript interfaces.
-   Separate business logic into custom hooks (e.g., `usePositionStatus`).
-   Use Supabase Row Level Security (RLS) policies mentally when designing data access.
-   **File Structure:** `app/`, `components/ui`, `components/business` (for complex domain widgets), `lib/`, `hooks/`.
-   **Language:** Code variables in English, UI text/Labels in Turkish.

# AI Behavior
-   Always check for "Document Lock" logic when writing status update functions.
-   When creating UI for forms, always implement validation with Zod schemas that match the DB constraints.
-   Prioritize "Scannability" in UI. Key metrics (Profit, Status) should be visible at a glance.
